## 实测

去噪 loss（CPU，T=50，27847 对）：
0.23(ep20) → 0.13(ep60) → 0.10(ep100) → 0.080(ep200)，平滑下降并趋稳。

五方对比（50 episode，seed 70000）：
| 策略 | 成功率 | 终末距离 |
|---|---|---|
| Expert (IK) | 90% | 67 mm |
| BC          | 56% | 143 mm |
| DAgger      | 86% | 68 mm |
| Diffusion   | 46% | 174 mm |
| PPO         | 88% | 64 mm |

DP 动作复现 MSE vs 专家（held-in 专家状态）：DP 0.33 | BC 0.12

## 结果判定：作为 Day 20 里程碑，成功
目标是"从零正确实现扩散策略 + 理解机制"，不是"打赢 BC"。判定依据：
1. 去噪 loss 从 0.23 平滑降到 0.08 → 网络确实在学去噪。
2. DP Reach 46%，远高于随机策略 ~0–2% → 在产生有目的的动作，不是乱走。
3. DP ≈ BC 档（同被协变量偏移卡在 ~50%）→ 符合"单模态任务上 DP 无优势"的预期。

DP 略低于 BC（46% vs 56%）也合理：DP 每步采样注入随机扰动，单模态任务上无收益、
反而让轨迹更抖；且仅 50 episode，46 与 56 差约 1.4 个标准误，部分是噪声。

## sanity check 的诚实说明
"DP 0.33 vs BC 0.12 同量级 → 正确"这句脚本打印是错的，0.33 ≈ 2.7×0.12。
但这不代表实现错——是该 check 设计得不公平：BC 用 MSE 回归，数学上输出条件均值 E[a|s]
（MSE 最优点估计），对专家动作的 MSE 天然最小；DP 输出的是从 p(a|s) 采的一个样，
样本 = 均值 + 分布离散度，结构上必然比均值离专家动作更远。拿采样器 MSE 比回归器 MSE
对采样器不公平。DP 实现是否正确，应看 loss 收敛 + 46% 有目的的 Reach，而非这个 MSE。

## 核心认知（面试可讲）
1. 扩散三段结构：加噪（固定，定义"数据→高斯"的桥）→ 去噪（学习，反着走桥）→
   生成（从新鲜高斯噪声出发反复去噪，雕出全新样本）。
2. 目标是生成而非还原：训练用(干净,加噪)对只为教会"去噪技能"，生成时丢掉原动作、
   对纯随机噪声施技 → 产生新动作；随机起点决定雕出分布里的哪一个。
3. 模型靠"学到的合法动作结构 + 观测 s 的约束"区分信号与噪声；单步分不开（高噪声几乎认不出），
   故迭代 50 步由粗到细逐步分离。
4. DP = 更强的策略类（能表达多峰 p(a|s)），但不解决协变量偏移；单模态 Reach 上是纯开销。
5. 代价：推理需 T 次网络前向（DP eval 比 BC 慢一两个数量级）→ 催生 DDIM / consistency 少步采样。
6. 回归 vs 采样：BC 输出均值（多峰会被平均成无效中值），DP 输出分布样本（可二选一）——
   这是 Day 21 多模态任务里 DP 胜出的根源。

## 工程要点
- 观测归一化用 buffer 随 state_dict 保存；动作本就 ~[-1,1]，无需额外归一化。
- 训练：随机抽噪声等级 t、自己造噪声 ε、加噪得 a_t、让网络回归 ε（纯监督 MSE）。
- 采样：从 randn 起，reversed(range(T)) 逐步去噪，每步 mean 公式去掉一小步噪声 +（除末步）撒少量随机。
- ε-prediction 优于直接预测干净动作：目标在所有噪声等级单位尺度，loss 更均衡。
- DP eval 慢（T 次前向/动作）：本日用 50 episode；想更快可减少 T 或用确定性采样。
