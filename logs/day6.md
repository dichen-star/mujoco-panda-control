## 学到

### 数学概念
- IK 没有解析解，必须用数值迭代
- Newton-Raphson 在 SE(3) 上：theta_{n+1} = theta_n + J^+ * error
- 雅可比伪逆 J^+ 在奇异附近爆炸 → 用阻尼最小二乘 DLS
- DLS: delta_theta = J^T (JJ^T + lambda^2 I)^-1 * error
- SE(3) 对数 log(T) = 位姿差 → 6 维旋量误差
- G^-1(theta) 系数公式中 cot(theta/2) 的处理

### 工程经验
- 阻尼系数 lambda = 0.01 在奇异和精度间平衡良好
- 单初值 IK 成功率有上限（约 80%），多次随机重启才是工业级标准
- max_iter 从 200 提到 500 可显著提升边界 case 成功率
- 关节限位 clip 防止 IK 飞出工作空间
- 收敛判据：位置 < 1e-4 m，旋转 < 1e-3 rad
- 7-DoF 冗余：多解，DLS 默认收敛到初值最近的解

### 多次重启策略
工业界 IK 算法（KDL、TRAC-IK、MoveIt）都用多次初值尝试：
1. 首选有意义的初值（HOME 姿态，远离奇异）
2. 失败后用随机关节角重启 3-4 次
3. 全部失败才算真正不可达

### IK 失败的真实原因
失败的 3 个案例（index 14, 57, 74）：
- 目标位姿采样到了 Panda 的深度奇异区域
- 即使工业级求解器在这种 corner case 也会失败
- 现实控制场景下目标是连续变化的，几乎不会卡在这种位置
